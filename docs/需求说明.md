# 加密货币合约市场监控告警 — 需求说明（初版）

## 1. 文档目的

描述「合约市场异动监控与告警」产品的首期功能范围：如何形成**观察列表**、如何监控**大盘动能**与**观察列表内标的**，以及在满足条件时如何**推送告警**。实现阶段以本文档为验收基线；交易所、符号口径、推送渠道可在配置中替换。

---

## 2. 名词与范围

| 名词 | 含义 |
|------|------|
| 观察列表 | 需要被**默认实时监控**的合约标的集合；列表内标的触发规则即产生告警候选。 |
| 异动信号 | 满足某条监控规则时产生的结构化事件（含标的、规则类型、关键数值、时间等）。 |
| 大盘动能标的 | 用于判断「市场整体是否异常放量」的代表性合约；列表**可配置扩展**。 |
| 山寨标的（本期） | 业务上指相对大盘代表币更偏「长尾」的合约；**实现默认**：观察列表内标的，且**不属于**大盘动能列表（可在配置中改为「观察列表内全部」）。 |

---

## 3. 产品目标

1. **自动维护观察列表**：从全市场数据中筛出可能异常活跃或拥挤的标的，降低人工盯盘成本。  
2. **双层监控**：板块级（少数代表合约 1m 成交量异动）+ 标的级（观察列表内爆仓、持仓量、成交量）。  
3. **实时告警**：规则触发后推送可读、可行动的告警信息。

---

## 4. 功能需求

### 4.1 观察列表

**FR-4.1-1** 系统维护一份观察列表；**列表内合约默认实时监控**。  

**FR-4.1-2** 列表须支持：自动入列、展示、**入列时间与入列原因**（枚举，便于审计与复盘）。  

**FR-4.1-3 自动入列规则（本期）**

1. **全市场持仓量 7 日涨幅榜**  
   - 对全市场可统计合约计算「持仓量（Open Interest）相对 7 日前同口径的变化率」。  
   - 取涨幅**排名前 15** 的标的加入观察列表（已在列则更新排名/刷新时间）。  

2. **持仓价值 / 代币市值**  
   - 计算 `持仓价值 / 代币市值`。  
   - 当比值 **> 0.8** 时加入观察列表。  

**FR-4.1-4** 两条规则**独立生效**；同一标的可同时携带多条入列原因。  

**FR-4.1-5（留白）** 出列规则、列表容量上限、同标的重复入列策略由后续版本补充。

---

### 4.2 大盘动能异常监控

**FR-4.2-1** 默认大盘动能标的：`BTCUSDT`、`ETHUSDT`、`XAUUSDT`、`CLUSDT`（以实际交易所可用符号为准）。  

**FR-4.2-2** 支持在配置中**增删**大盘动能列表（与默认列表合并方式见配置项 `macro.mergePolicy`）。  

**FR-4.2-3** 在 **1m** K 线维度监控成交量；相对基线放大至 **`volumeMultMin`～`volumeMultMax` 倍**（默认 20～50）时触发**大盘动能异常**告警。  

**FR-4.2-4** 基线算法须固定、可复现（见附录 A.3）；边界是否包含端点由配置 `macro.boundaryInclusive` 固定。

---

### 4.3 观察列表内标的异动（标的级）

以下规则作用于**已在观察列表**且符合「山寨范围策略」的标的（默认：不在大盘动能列表中）。

**FR-4.3-1 爆仓**  
- 当单条事件或聚合窗口内爆仓规模 **> `liquidation.minNotional`**（默认 100,000，计价单位见配置）时推送告警。  

**FR-4.3-2 持仓量入列后高点突破**  
- 自标的**进入观察列表时刻**起维护 OI **历史最高**（仅统计入列后）。  
- 当前 OI **严格大于**该历史最高时触发（首次入列当根是否计入见附录 A.4）。  

**FR-4.3-3 持仓量短时涨幅**  
- 在 **5m、15m** 周期上监控 OI 变化率；落在 **`oiSpike.minPct`～`oiSpike.maxPct`**（默认 10%～20%）区间内触发。  
- 「落在区间内」指达到下限即告警，或同时要求不超过上限由 `oiSpike.mode` 定义（见配置表）。  

**FR-4.3-4 成交量短时涨幅**  
- 在 **5m、15m** 上监控成交量相对基线的涨幅；**≥ `volumeSpike.minPct`**（默认 100%）时触发（与「大于」二选一由 `volumeSpike.boundary` 固定）。  

---

### 4.4 告警与推送

**FR-4.4-1** 告警最小字段：标的、规则类型、触发时间（统一 UTC 或统一交易所时区）、关键数值、入列原因（若适用）、简短说明。  

**FR-4.4-2** 对同一标的、同一规则支持**冷却时间**或合并推送（`alert.cooldownSec`），避免刷屏。  

**FR-4.4-3（留白）** 推送渠道（Telegram / 钉钉 / Webhook 等）不在本期文档强制指定。

---

## 5. 可配置项与默认值

建议集中为一份 `config`（JSON/YAML/环境变量均可），下列为**逻辑键名**与**首期默认**。

### 5.1 总表

| 配置键 | 含义 | 默认值 | 备注 |
|--------|------|--------|------|
| `exchange` | 数据源交易所 | `binance` | 可扩展 okx、bybit 等 |
| `marketType` | 合约类型 | `usdt-m` | U 本位 / 币本位等 |
| `watchlist.oi7d.topN` | 7 日 OI 涨幅榜入列数量 | `15` | |
| `watchlist.oiValueToMcap.threshold` | 持仓价值/市值入列阈值 | `0.8` | 大于该值入列 |
| `watchlist.scanIntervalSec` | 全市场扫描周期 | `300` | 可按 API 限额调整 |
| `macro.symbols` | 大盘动能标的列表 | 见 4.2.1 | 可覆盖或追加 |
| `macro.mergePolicy` | 与用户扩展列表合并策略 | `union` | `union` / `replace` |
| `macro.interval` | 动能 K 线周期 | `1m` | |
| `macro.volumeMultMin` | 成交量倍数下限 | `20` | |
| `macro.volumeMultMax` | 成交量倍数上限 | `50` | |
| `macro.baselineWindow` | 1m 成交量基线窗口（根数） | `60` | 约 1 小时 |
| `macro.baselineMethod` | 基线算法 | `sma` | `sma` / `median` |
| `macro.boundaryInclusive` | 倍数区间是否包含端点 | `true` | 20、50 是否算触发 |
| `alt.scope` | 山寨规则作用范围 | `not_in_macro` | `not_in_macro` / `all_in_watchlist` |
| `liquidation.minNotional` | 爆仓告警最小名义金额 | `100000` | 单位见 `liquidation.notionalUnit` |
| `liquidation.notionalUnit` | 爆仓金额计价 | `USDT` | |
| `liquidation.aggregateWindowSec` | 爆仓聚合窗口（0 表示按单笔） | `0` | |
| `oiSinceEntry.useMark` | OI 是否用标记价格口径 | `false` | 与数据源一致即可 |
| `oiSpike.intervals` | OI 短时监控周期 | `["5m","15m"]` | |
| `oiSpike.minPct` | OI 涨幅下限（小数） | `0.10` | 10% |
| `oiSpike.maxPct` | OI 涨幅上限（小数） | `0.20` | 20% |
| `oiSpike.mode` | 触发语义 | `gte_min_and_lte_max` | 见下表 |
| `volumeSpike.intervals` | 成交量监控周期 | `["5m","15m"]` | |
| `volumeSpike.minPct` | 成交量相对基线最小涨幅 | `1.00` | 100% |
| `volumeSpike.baselineMethod` | 成交量基线 | 同 macro 或独立 | 建议与对应周期一致 |
| `volumeSpike.boundary` | 比较边界 | `gte` | `gt` / `gte` |
| `alert.cooldownSec` | 同标的同规则冷却秒数 | `60` | `0` 表示不冷却 |
| `time.standard` | 日志与告警时间标准 | `UTC` | |

### 5.2 `oiSpike.mode` 取值说明

| 值 | 含义 |
|----|------|
| `gte_min` | 涨幅 ≥ min 即告警（可能远大于 20%） |
| `gte_min_and_lte_max` | min ≤ 涨幅 ≤ max 才告警（贴合「10%～20%」字面区间） |
| `gte_min_lt_max` | min ≤ 涨幅 < max |

**默认建议**：若产品语义强调「巨量但仍在 10%～20% _band_」，用 `gte_min_and_lte_max`；若强调「至少 10%」，用 `gte_min`。

## 6. 非功能需求（概要）

| 类别 | 要求 |
|------|------|
| 实时性 | 在方案中声明目标延迟（如 P95）；1m 规则依赖行情聚合时效。 |
| 可靠性 | 数据源故障时降级、明显不可信数据不触发或打标「低置信」。 |
| 可复现 | 同一配置 + 同一原始数据快照 → 指标与触发结果一致。 |
| 可观测 | 入列、触发、冷却丢弃原因可查询日志。 |

---

## 7. 本期范围外

- 观察列表出列与容量上限。  
- 精确「山寨」标签源（板块、市值分层等）。  
- 非 USDT 本位、跨所套利、资金费率联动等扩展规则。

## 附录 A：数据字段与计算公式（实现参考）

以下以 **Binance U 本位合约**为**默认假设**；其他交易所需映射等价字段。

### A.1 建议统一字段

| 字段 | 说明 |
|------|------|
| `symbol` | 合约代码，如 `BTCUSDT` |
| `openInterest` | 持仓量（张数或币数量，全链路统一一种） |
| `openInterestValue` | 持仓名义价值（若 API 直接提供） |
| `markPrice` / `indexPrice` | 标记价、指数价 |
| `volume` | K 线成交量（单位与 API 一致） |
| `marketCap` | 标的对应现货代币市值（需指定数据源：CoinGecko/CMC/自建） |
| `liquidation.qty` / `liquidation.price` | 爆仓侧用于折算名义金额 |

### A.2 入列指标公式（建议）

**7 日 OI 涨幅**

\[
\text{oiReturn7d} = \frac{\text{OI}(t) - \text{OI}(t-7d)}{\max(\epsilon, \text{OI}(t-7d))}
\]

- `t-7d` 取最近可用快照（整点对齐或按日 UTC 0 点，**选定一种并写死**）。  
- \(\epsilon\) 为防止除零的极小正数；若历史 OI 缺失则该标的本轮不参与排名或标记数据不足。

**持仓价值 / 市值**

\[
\text{ratio} = \frac{\text{openInterestValue}}{\max(\epsilon, \text{marketCap})}
\]

- 若仅有 OI 与标记价：`openInterestValue ≈ OI × markPrice`（单位换算一致）。  
- `marketCap` 与合约 `symbol` 的映射关系需维护（如 `1000PEPEUSDT` → `PEPE`）。

### A.3 大盘 1m 成交量倍数

设当前根成交量为 \(V_t\)，基线为前 `macro.baselineWindow` 根（不含当前根）的均值或中位数 \(B_t\)。

\[
\text{mult} = \frac{V_t}{\max(\epsilon, B_t)}
\]

触发条件（端点由 `macro.boundaryInclusive` 控制）：

\[
\text{volumeMultMin} \le \text{mult} \le \text{volumeMultMax}
\]

### A.4 入列后 OI 高点

- 记 `entryTime` 为标的进入观察列表的时间。  
- 维护 `maxOI = max{ OI(\tau) | \tau \ge entryTime }`（可与 K 线收盘对齐）。  
- 触发：当前 `OI > maxOI`（突破）；更新 `maxOI`。  
- **首次入列**：可选策略（配置 `oiSinceEntry.includeFirstBar`）：`true` 表示入列后第一根 K 线即可参与更新高点；`false` 表示从第二根开始，避免入列瞬间噪声。

### A.5 5m / 15m OI 与成交量涨幅

对周期 \(k \in \{5m,15m\}\)，设窗口首尾 OI 为 \(\text{OI}_{start}, \text{OI}_{end}\)：

\[
\text{oiChgPct} = \frac{\text{OI}_{end} - \text{OI}_{start}}{\max(\epsilon, |\text{OI}_{start}|)}
\]

成交量涨幅：对窗口内成交量合计或首尾差分，与「基线窗口」对比得到 `volumeChgPct`（与 `volumeSpike.baselineMethod` 一致即可）。

### A.6 爆仓 100k

- 若数据源为单笔爆仓：单笔折算名义金额（建议 USDT）> `liquidation.minNotional` 即触发。  
- 若为聚合流：在 `liquidation.aggregateWindowSec` 内求和后比较阈值。

## 附录 B：验收检查清单

1. 展示观察列表及每条**入列原因**与时间。  
2. 周期性产出「7 日 OI 涨幅 Top N」并入列。  
3. `持仓价值/市值 > 阈值` 的标的入列。  
4. 大盘动能列表默认包含四类标的，且可配置增删。  
5. 1m 成交量倍数落在配置区间内时产生大盘动能告警。  
6. 观察列表内标的（按 `alt.scope`）：爆仓、入列后 OI 新高、OI 5m/15m、成交量 5m/15m 规则可分别触发，且冷却策略生效。
